# Basiliskode Brand System

This document translates the Basiliskode mini brand book into practical rules for repositories, README pages, reports, websites, screenshots, and AI-generated assets.

## Core identity

| Element | Rule |
|---|---|
| Brand name | `basiliskode` |
| Descriptor | `AI SYSTEMS` |
| Visual idea | A technical manual for intelligent systems |
| Symbol | Modular basilisk-infinity mark |
| Personality | Precise, vigilant, silent, recursive, controlled |
| Avoid | Generic SaaS, purple AI gradients, glossy startup language |

Basiliskode should feel like a **classified operating manual for adaptive AI systems**: engineered, calm, technical, and slightly dangerous.

## Philosophy

Use these words as the brand spine:

- **Vigilance** — systems that observe and retain signal.
- **Wisdom** — memory that compounds into judgment.
- **Cycle** — learning is recursive, not linear.
- **Transformation** — raw interaction becomes durable knowledge.
- **Control** — local-first, auditable, no vendor lock-in.
- **Intelligence** — quiet capability over loud claims.

## Color system

| Token | Hex | Usage |
|---|---:|---|
| Mint | `#7EF0C2` | Primary accent, logo, active states, section numerals |
| Deep black | `#080908` | Main background |
| Bone gray | `#E2E5E5` | Primary readable text |
| Slate gray | `#2A2D2C` | Borders, grid lines, secondary surfaces |

Rules:

- Use **one accent only**: mint.
- Keep backgrounds near-black, never pure `#000000`.
- Use thin technical grid lines instead of soft shadows.
- Do not use violet, blue-purple, neon gradients, or generic AI glow.

## Typography

| Role | Font | Usage |
|---|---|---|
| Primary | Space Mono | Brand headers, terminal text, labels, metadata, code-like UI |
| Secondary | IBM Plex Sans | Longer readable paragraphs and support text |

Rules:

- Prefer lowercase brand text: `basiliskode`.
- Use uppercase for system labels: `AI SYSTEMS`, `MANUAL`, `APPLICATIONS`.
- Keep headings tight, technical, and direct.
- Avoid playful or rounded fonts.

## Layout language

The layout should look assembled from a technical grid.

- Use visible module boundaries: 1px grid lines, panels, crosshair marks.
- Use section numbers: `01.`, `02.`, `03.`.
- Use small terminal-style captions: `> basiliskode://manual`.
- Prefer asymmetric technical compositions over centered marketing blocks.
- Make whitespace feel measured, not empty.

## Logo usage

The basilisk-infinity mark represents:

- infinite learning loops,
- memory cycles,
- vigilance,
- structured transformation.

Allowed versions:

- mint on deep black,
- black on mint,
- monochrome white/bone,
- reduced icon for small spaces.

Forbidden usage:

- changing proportions,
- changing colors outside the palette,
- adding glow/effects,
- rotating the mark,
- using only outlines,
- invading the clear-space area.

## Copy voice

The voice is technical, terse, and confident.

Write like this:

> Local memory for agents that need to remember what matters.

> SQLite-backed facts, trust scoring, lifecycle control, and zero vendor lock-in.

Avoid this:

> Unleash next-gen AI memory with seamless intelligent experiences.

Rules:

- Prefer concrete nouns: facts, sessions, retrieval, trust, archive, export.
- Prefer verbs with weight: store, retrieve, reinforce, expire, inspect.
- Avoid startup filler: seamless, unleash, elevate, revolutionary, next-gen.
- Say what the system does. Do not hype what it might become.

## UI direction

For websites, README visuals, benchmark reports, and dashboards:

- Background: deep black.
- Text: bone gray.
- Accent: mint.
- Components: terminal panels, technical tables, thin borders.
- Motion: restrained; cursor blinks, grid scans, subtle status pulses.
- Icons: geometric, line-based, functional.

Do not use:

- glassmorphism,
- soft pastel cards,
- emoji-first UI,
- generic three-card SaaS sections,
- purple-blue AI branding.

## README / GitHub application

Use a technical-manual structure:

```md
> basiliskode://memory-etch

# memory-etch

LOCAL-FIRST MEMORY FOR AI AGENTS

01. STORE FACTS
02. RETRIEVE SIGNAL
03. REINFORCE TRUST
04. ARCHIVE NOISE
```

For badges and callouts, prefer bracketed labels:

```md
[ LOCAL-FIRST ] [ SQLITE ] [ FTS5 ] [ HRR ] [ MCP ] [ NO VENDOR LOCK-IN ]
```

## Asset checklist

Before publishing any Basiliskode visual, verify:

- [ ] Uses `basiliskode` lowercase.
- [ ] Uses `AI SYSTEMS` as descriptor when brand context is needed.
- [ ] Uses only mint, deep black, bone gray, slate gray.
- [ ] Uses Space Mono / IBM Plex Sans or close equivalents.
- [ ] Uses grid/manual/terminal structure.
- [ ] Avoids purple AI gradients and generic SaaS styling.
- [ ] Feels precise, controlled, and system-level.
